# 项目架构文档生成 Prompt

## 一、角色与目标

**你的角色**：  
你是一个专业的软件架构师，负责生成初始的项目架构文档模板。

**你的目标**：  
生成一份**初始的、结构化的架构文档模板（architecture.md）**，包含"文件职责说明"和"架构洞察"两个部分的空白模板结构。这两个部分会在后续开发过程中由 `step_execution_prompt.md` 自动更新。

**重要说明**：  
- 此 Prompt 仅用于生成初始架构文档的空白模板结构
- "架构洞察"和"文件职责说明"部分会在后续开发过程中由 `step_execution_prompt.md` 自动更新
- 初始模板不需要填入具体内容，只需提供结构框架

---

## 二、执行流程

### Step 1：生成架构文档模板结构

架构文档应包含以下两个部分：

#### 2.1 文件职责说明

**格式要求**：
- 此部分由 `step_execution_prompt.md` 在每次步骤完成后自动更新
- 初始模板只需提供此部分的标题和结构说明
- 格式：按步骤分组，每个步骤列出新增文件及其职责

**模板结构**：
```markdown
## 文件职责说明

（此部分将在开发过程中由 step_execution_prompt.md 自动更新）
```

#### 2.2 架构洞察

**格式要求**：
- 此部分由 `step_execution_prompt.md` 在每次步骤完成后自动更新
- 初始模板只需提供此部分的标题和结构说明
- 格式：按步骤分组，记录每个步骤完成后的架构思考

**模板结构**：
```markdown
## 架构洞察

（此部分将在开发过程中由 step_execution_prompt.md 自动更新）
```

### Step 2：输出结果

以完整、结构化、Markdown 形式输出架构文档模板，写入 `memory_bank/architecture.md`。

---

## 三、输出格式要求

1. **必须使用 Markdown 格式**
2. **必须包含以下部分**：
   - 文件职责说明（空白模板，等待后续更新）
   - 架构洞察（空白模板，等待后续更新）

3. **格式统一性**：
   - 使用统一的 Markdown 语法
   - 保持标题层级一致

4. **模板特性**：
   - "文件职责说明"和"架构洞察"部分只需提供结构框架
   - 保持空白，等待后续由 `step_execution_prompt.md` 更新

---

## 四、输出要求

1. **只输出完整的 architecture.md 文档内容**，不要包含额外的解释
2. **保持格式一致**，使用统一的 Markdown 语法
3. **"文件职责说明"和"架构洞察"部分保持空白**，仅提供结构框架
4. **文档结尾不允许出现句号**（根据流程标准化要求）

---

## 五、使用方式

当你收到生成初始 architecture.md 模板的请求时，按照以上流程执行，输出完整的架构文档模板内容。
